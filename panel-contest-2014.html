Kivy Programming Contest #2

It's time for Kivy's second programming contest, a chance for Kivy users both new and old to show off their skills and compete for awesome prizes. Competing programs will be judged on a range of criteria, so don't be afraid to dive in! The theme of this contest is education. This is a broad theme, you could make anything from a maths game, to a handwriting tutor, to a turtle graphics toy...or anything else you can imagine. Registration will open 2 weeks before the official start date, but you can enter any time up to the final deadline. Give it a go!

Rules

These rules are a general guideline, exact rules and terms will be posted before the contest starts. The goal is to show the world some of the great things that be built quickly with Kivy while having fun, and to give more people an opportunity to try Kivy's cool features. The general guidelines are:
  • Keep it family friendly.
  • Make it fun and attractive, show how your Kivy app can stand out from the crowd.
  • Entries can be from individuals or from teams.
  • No prior code - build your full app during the competition month.
  • Don't forget the theme of education!
The following are the specific requirements for your applications, relating to how they will be tested and judged.
  1. You the participant (individual or team) must create an app with the education theme. Don't feel constrained, you can make anything with this general theme...be creative!
  2. You officially enter the contest using the application form, which we'll open on April 2nd. You can enter at any time up until the deadline, but the earlier the better!
  3. You must submit the game developed using Kivy by uploading it to a public Github repository before the official deadline, and including your repository name in your official entry. Don't leave this until the last day! It's easy to setup Git and create a repository with your code in it.
  4. You can use Kivy and standard python libraries. Use of external Python libraries is also tolerated provided they are either pure python and available via pip, or have pre-existing recipes in Kivy's build tools for the platforms you are targeting.
  5. You can target specific platforms or hardware features, but must include clear setup instructions in your README. Regardless, we encourage taking advantage of Kivy's cross-platform potential rather than writing for a specific platform.
  6. You must respect the standard Kivy Application layout as defined in Kivy documentation, and marks will be given for following good Kivy and Python conventions. The game must run on latest stable Kivy (1.8.0), with the command "python main.py".
  7. Usage of custom OpenGL is not allowed. Your application must be compatible OpenGL ES 2.0.
  8. The game should be tested before hand. A failure to start will result in negative points.
  9. Any adult or offensive content is forbidden. The game should be child friendly.
  10. Entries will be scored and prizes awarded by Panel of judges who are excluded from participating. Scoring will be based on several criteria as defined below.
  11. Your app will be scored based on the version in its official repository at the instant of the deadline.
  12. Your can choose the license for your program, but we strongly encourage using an open source license so others can change, share and learn from your code. Our only requirement is that your Github repository be public so that we can test and judge your entry.
Every entry will be scored with a mark out of 20 for each of the following criteria. The best entries are generally those that do well in all of these:
  • Idea (funny/addictive, originality/uniqueness)
  • Smooth gameplay (stutters, hiccups, game logic)
  • Simplicity / usability
  • Good use of Kivy logic and general style
  • Production quality (graphics, sound, polish)
